There are two ways a machine like a phone, computer, or tablet can be on the network: it can be a "client" or a "server". When you point your web browser to a page, you're using your machine as a client. Whatever happens in that browser on your machine is called "client side".

In order to have a decentralized network, however, we also need "servers", which are just computers that host the files that we read on the clients. Pretty much any device capable of being a client can also be a server(phone, tablet, laptop, desktop, Raspberry Pi, "internet of things" devices, routers, etc.). Also, there are numerous ways to get your files out on the World Wide Web without running your own server for free, using varous free cloud-based services. The purpose of this file is to help you get at least one of these numerous choices up and running as easily and quickly as possible.
